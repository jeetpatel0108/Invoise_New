   ✓ Each bill gets unique number: INV-YYYYMMDD-XXX-YYYY
   ✓ Not just "Invoice #1" anymore
   ✓ Easy to identify and reference
   
   Examples:
   User 1, May 21 → INV-20250521-001-0001
   User 1, May 21 → INV-20250521-001-0002 (next bill same day)
   User 1, May 22 → INV-20250521-001-0001 (resets per day)
   User 2, May 21 → INV-20250521-002-0001 (different user)
   
   What changed:
   - Database: Added bill_number column to invoices table
   - Function: generate_bill_number() creates unique numbers
   - PDF: Shows "Bill No: INV-20250521-001-0001"
   - Dashboard: Shows bill numbers instead of IDs
   - Invoice History: Shows bill numbers
   - Success Page: Shows "Bill No INV-..."

❓ FAQ:

Q: Will old invoices still work?
A: Yes! They'll show fallback number "INV-{id}" until regenerated

Q: How do I enable SMS sending?
A: Set FAST2SMS_API_KEY environment variable with your API key

Q: Can I customize bill number format?
A: Yes! Edit generate_bill_number() function in app.py

Q: What if I need to reset OTP?
A: User can click "Resend OTP" button on forgot password page

Q: Is OTP secure?
A: Yes! Hashed with SHA256, attempt limiting, expiry timer
